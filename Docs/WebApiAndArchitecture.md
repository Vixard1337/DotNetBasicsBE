# Web API, architektura i wzorce

## 17. ASP.NET Core Web API
- Kontrolery, routing, model binding.
- `IActionResult` vs `IResult`.
- Minimal APIs.
- Middleware.

## 18. REST i CRUD
- REST, CRUD, metody HTTP.
- Statusy: 200, 201, 204, 400, 404, 500.
- Idempotencja.

## 19. HTTP/HTTPS
- Request/Response, nagłówki, JSON.
- TLS/SSL.

## 20. Walidacja
- Data Annotations, FluentValidation, `ModelState`.

## 21. Swagger / OpenAPI
- Dokumentacja i testowanie endpointów.

## 22. Dependency Injection (DI)
- IoC, `AddScoped`, `AddTransient`, `AddSingleton`.

## 23. MediatR
- Mediator pattern, Request/Response, Command/Query, Handlers, Pipeline Behaviors.

## 24. CQRS
- Rozdzielenie zapisu i odczytu.

## 25. MVC
- Model–View–Controller, różnice MVC vs Web API.

## 26. Wzorce projektowe
- Creational: Singleton, Factory Method, Abstract Factory, Builder.
- Behavioral/Structural: Strategy, Repository, Unit of Work, Observer, Decorator, Adapter, Facade, Mediator.

## 27. Zasady projektowe
- SOLID, DRY, KISS, YAGNI.

## 31. Architektura aplikacji
- Warstwy: API, Application, Domain, Infrastructure.
- Clean Architecture (rola każdej warstwy).
